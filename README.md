# Resumer

作業の「中断」を失敗ではなく **データ** として扱い、次の行動への **復帰を支援する** アプリケーション。

- 中断理由・状況を即時に記録し、復帰しやすい状態をつくる
- 「表層理由」と「構造要因」を分けて分析し、再発を防ぐ

## 🧠 コンセプト
- 中断は「怠惰」ではなく「状態遷移」
- 復帰できない原因は、意志ではなく構造にある
- 行動ログよりも「中断の瞬間」に価値がある

## 🎯 解決したい課題
- 中断理由を後から思い出せない
- 再開時に「何から手をつければいいか」分からない
- 中断が続き自己評価が下がる
- タスク管理が逆にプレッシャーになる

## 👤 想定ユーザー / シナリオ
- 個人開発者・ナレッジワーカー・学生
- 30–120分の集中作業を繰り返し、中断が多いリモート環境
- すぐ復帰したいが、再開までに迷子になりがち

## ✨ 主な機能（MVP）
### 1. 中断キャプチャ
- 中断理由（選択式＋自由記述）
- 状態（疲労 / 集中切れ / 外部要因 など）
- 気分・思考（任意）
> 正確さより「即時性」を優先

### 2. 復帰ガイド生成
- 前回どこで止まったか
- 次にやるべき「最小の一手」
- 過去の類似ケースの復帰ヒント

### 3. 中断パターン分析
- よくある中断理由
- 時間帯・作業種別との相関
- 復帰できたケース / できなかったケース

## 🧩 設計思想
- **中断原因 ≠ 真因**: 表層理由と構造要因を分離して扱う
- **入力負荷の最小化**: 選択式中心、空白保存可、後から補足可能
- **イベント中心モデル**: 中央のタスクではなく「中断イベント」を核にする

## 📊 データとプライバシー
- 収集: 中断理由・状態・時刻・作業メモ（任意）
- 保存: 端末ローカル（初期）、将来的にクラウド同期を検討
- 配慮: 削除機能、保存期間の設定

## 🧪 成功指標
- 再開までの平均時間の短縮
- 中断後の復帰率向上
- 主観的な自己効力感/満足度の改善

## 🏗 アーキテクチャ（MVP想定）
- フロントエンド: React Native / Expo
- データモデル: 中断イベント中心（理由・状態・時刻・メモ・タスク紐付け）
- ストレージ: 端末内ストア（AsyncStorage等）、将来は同期層を追加
